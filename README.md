# Remarq

Pin-and-comment feedback overlay for design prototypes and web apps. Let your team leave contextual feedback directly on the UI.

## Features

- **Click to pin** — drop comment pins anywhere on the page
- **Smart target detection** — auto-anchors to nearest meaningful element
- **Thread-based** — replies, resolve/unresolve, delete
- **Keyboard shortcut** — press `C` to toggle, `Esc` to cancel
- **Popover-aware** — comments inside modals/popovers re-appear when reopened
- **All Pages view** — see every comment across your project in one sidebar
- **Auto z-index** — overlay detects highest z-index and sits above everything
- **Environment modes** — dev-only, personal, or public (all environments)
- **SSR-safe** — works with Next.js App Router

## Quick Start

### 1. Install

```bash
npm install -g remarq
```

### 2. Initialize

```bash
cd [my-project]
remarq init
```

This creates:
- `.remarq/` — comment storage directory
- `api/remarq/route.ts` — Next.js API route
- `components/remarq-wrapper.tsx` — pre-configured wrapper component
- Updates `.gitignore` based on mode

### 3. Add one line to your layout

```tsx
// app/layout.tsx
import { RemarqWrapper } from "@/components/remarq-wrapper";

export default function Layout({ children }) {
  return (
    <html>
      <body>
        <RemarqWrapper>{children}</RemarqWrapper>
      </body>
    </html>
  );
}
```

### 4. Start your dev server

```bash
npm run dev
```

Press `C` on any page to start commenting.

## Init Modes

```bash
remarq init            # Default (--dev)
remarq init --dev      # Dev + staging only
remarq init --personal # Local dev only, fully private
remarq init --public   # All environments including production
```

| Mode | Active in | Comments in git | Route in git | Env override |
|------|-----------|----------------|--------------|-------------|
| `--dev` | Dev + staging | No | Yes | `NEXT_PUBLIC_REMARQ=true` to force on |
| `--personal` | Local dev only | No | No | — |
| `--public` | All environments | Yes | Yes | `NEXT_PUBLIC_REMARQ=false` to disable |

## CLI Commands

```bash
remarq init [--dev|--personal|--public]   # Set up remarq
remarq remove                              # Remove remarq
remarq help                                # Show help
```

### `remarq init`

| What it does | Details |
|---|---|
| Creates `.remarq/` | Comment storage directory |
| Updates `.gitignore` | Based on mode |
| Detects framework | Next.js auto-setup |
| Creates API route | `api/remarq/route.ts` |
| Creates wrapper | `components/remarq-wrapper.tsx` with env guard |

### `remarq remove`

Removes API route, cleans `.gitignore`. Optionally delete `.remarq/` and wrapper manually.

## How It Works

Comments are stored as JSON files in `.remarq/`:

```
.remarq/
├── home.json
├── about.json
└── dashboard--settings.json
```

The wrapper auto-detects the current page from `usePathname()` and loads the corresponding comments.

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `C` | Toggle comment mode |
| `Escape` | Cancel unsaved comment / exit comment mode |
| `Enter` | Submit comment |

## Components

### `<RemarqWrapper>`

Generated by `remarq init`. Wraps your layout with the provider and all UI components. Includes environment guard based on init mode.

### `<RemarqProvider>`

Core context provider. Use directly for custom setups:

```tsx
<RemarqProvider pageId="my-page" storage={adapter}>
```

### `<CommentOverlay>`

Captures clicks in comment mode. Auto-detects z-index to sit above popovers and modals.

### `<CommentToggle>`

Floating button (bottom-right) with unresolved count badge.

### `<CommentSidebar>`

Right panel with two tabs:
- **This Page** — comments on the current page
- **All Pages** — every comment across the project. Click to navigate.

## Hooks

```tsx
import { useRemarq, useComments, useCommentMode } from "remarq";

const { threads, user, addThread, addReply, resolveThread } = useRemarq();
const { openThreads, resolvedThreads, unresolvedCount } = useComments();
const { commentMode, toggleCommentMode, sidebarOpen, toggleSidebar } = useCommentMode();
```

## Storage Adapters

### Default

No config needed. `remarq init` sets up the API route that reads/writes `.remarq/` files.

### localStorage

```tsx
import { localStorageAdapter } from "remarq/adapters/localStorage";
<RemarqProvider pageId="my-page" storage={localStorageAdapter}>
```

### Custom REST API

```tsx
import { createRestAdapter } from "remarq/adapters/rest";
<RemarqProvider pageId="my-page" storage={createRestAdapter("/api/my-comments")}>
```

### Custom Adapter

```tsx
const myAdapter = {
  async load(pageId) { /* return threads */ },
  async save(pageId, threads) { /* persist */ },
};
<RemarqProvider pageId="my-page" storage={myAdapter}>
```

## Target Detection

Remarq auto-detects meaningful elements when placing comments:

1. `data-comment-target="id"` — explicit anchor
2. Elements with `id` or `aria-label`
3. Semantic HTML (`section`, `nav`, `aside`)
4. Visual panels (scrollable, bordered, shadowed)

Pins stored as percentages relative to the target — they follow on scroll/resize. Pins inside popovers re-appear when reopened.
